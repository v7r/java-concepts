## 1️⃣ Universal Prefix Sum Mental Checklist

Whenever you see a subarray problem, ask **in this exact order**:

### ✅ Step 1: Is it a *contiguous* subarray?

* Yes → prefix sum might apply
* No → prefix sum does NOT apply

### ✅ Step 2: Can the condition be written as

```
subarray_sum(l → r) = f(k)
```

If yes, rewrite using prefix sums:

```
P[r] - P[l - 1] = f(k)
```

### ✅ Step 3: Rearrange

```
P[l - 1] = P[r] - f(k)
```

👉 **This tells you what to store and what to query**

If you can do Step 3, prefix sum + hashmap is guaranteed.

## 3️⃣ Canonical Reusable Code Template (Language-Agnostic)

This template works for **80% of prefix sum problems**.

```text
initialize prefixSum = 0
initialize map = { 0 : 1 }
initialize answer = 0

for each element x in array:
    prefixSum += x

    target = prefixSum - k
    if target exists in map:
        answer += map[target]

    map[prefixSum] += 1

return answer
```

### 🔑 Why this order matters

* **Count first**
* **Insert later**

This ensures subarrays end at the current index.

## 6️⃣ Sliding Window vs Prefix Sum (Quick Decision Rule)

| Condition               | Use            |
| ----------------------- | -------------- |
| All numbers positive    | Sliding window |
| Zeros / negatives exist | Prefix sum     |
| Counting subarrays      | Prefix sum     |
| Longest subarray        | Prefix sum     |
| Exact sum               | Prefix sum     |
